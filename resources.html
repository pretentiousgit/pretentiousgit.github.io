Software

  • VSCode
    • A text editor intended for programming computers
    • Not Co-Pilot. Although generative LLMs can be supportive in developing code, they are extremely likely to generate complex bugs that take a while to fix. For this class, it's simpler not to use it.
  • Extensions
    • Liveserver by Ritwick Dey
      • This adds a button to your interface that allows you to launch a Node server and test your web pages without running your own from the terminal/command line.
      • There is support built into VSCode for this now, if you prefer
    • Prettier
      • Prettier is an opinionated code formatter
      • This means if you right-click and select "format code" it will reformat your code to be clearer
    • ES6 String Literals extension by Tobermory
      • Allows you to see string literals with HTML syntax highlighting

    • Error Lens by "Alexander"
      • Strictly optional but sometimes helpful to show you what's wrong in JS

  • Node.js
    • Install the most recent "LTS" package. It should be 20.18 or higher, but not 22+ as of yet.
    • This is the "runtime" that lets us use Javascript on the server side, ie: without a browser

  • Github Desktop
    • This is helpful for storing code "repositories" and backing your work up to Github.
    • Downside: you may need to learn Git. We have a single lecture on this.
